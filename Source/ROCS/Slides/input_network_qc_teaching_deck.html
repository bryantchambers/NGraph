Input and Network QC for Ancient Metagenomic WGCNA

A grounded walkthrough using current repository outputs

Project practical

2026-05-19

Scientific Question

Can we build a stable, biologically interpretable microbial network from damaged ancient DNA?

Key practical challenge:

The strongest input PC is associated with detected taxa and read depth. We need to separate technical/preservation structure from biological structure before trusting downstream WGCNA modules and HMM states.

Repository Map

The main pipeline remains separate from contained QC analyses. InputQC and networkQC are decision-support folders.

What Each QC Area Answers

Rarefaction Caveat

Bootstrap Stability

scripts/02b_wgcna_stability.R

# lines 82-88: resample samples within each training core
idx <- sample(seq_len(nrow(dat)), size = nrow(dat), replace = TRUE)
dat[idx, , drop = FALSE]
# lines 143-154: match each original module to best bootstrap module
jaccard = intersection(original, bootstrap) / union(original, bootstrap)

Bootstrap stability asks whether module membership survives sample perturbation, not whether a module is biologically important.

HMM State Discovery

scripts/03_hmm_states.R

# lines 4-10
1. residualise eigengenes within core
2. PCA on training cores; project validation/all samples
3. fit K = 2..5 HMMs
4. score held-out validation core
5. select K using BIC + held-out likelihood/stability

Hybrid logic:

  • train BIC narrows plausible K,
  • held-out log likelihood picks predictive K,
  • transition stability penalizes noisy state switching.

What We Trust Now

  1. The input has a real detection/depth axis.
  2. The low-detection samples are not random noise.
  3. Rarefaction does not show broad undersequencing in the processed matrix.
  4. ST8 has a specific 90-110 kya low-detection issue.
  5. WGCNA parameter choice improved substantially over baseline.
  6. exp3 is the current best network setting.

How to Recognize Bad Analysis Signatures

Warning signs:

  • PC1 almost perfectly tracks read depth or detected taxa.
  • Grey module is huge and biologically tempting.
  • Module stability depends on a single core or age interval.
  • Rarefaction/Chao1 are interpreted after singleton-removing filters without caution.
  • Network plots are treated as module construction evidence.

Final Recommendation

Use exp3: power=12, deepSplit=3, mergeCutHeight=0.25, minModuleSize=20.

Rationale:

  • lowest grey burden among top candidates,
  • best integrated full-eval / kME / topology score,
  • best core-balance Jaccard among top candidates,
  • no kME/topology warning flags.

Continue treating input depth as a standing caveat until the ST8 removal sensitivity finishes.
